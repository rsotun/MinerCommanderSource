Настройки клиента расположены в файле minercommander.xml
--------------------------------------------------------------
<GPU type="AMD" ram="8" name="RX570" />
Type - тип видеокарты (пока не применяется )
ram - размер памяти карт на фермах(пока не применяется )
name - название фермы, по этому названию будет браться строка для батника.
--------------------------------------------------------------------------
<commander reftime="30" waitmsi="False" urlxml="http://xxxsite.ru/minerset.xml" laststr="" />
reftime - время через которые программа повторяет запрос на сервер в минутах.
urlxml - путь к XML файлу в котором необходимо брать актуальную строку батника,
например https://moisait.ru/batniki.xml, можно использовать айпи адрес вместо домена
waitmsi - дождаться запуска Msi Afterburner и только потом запускать майнер.(пока не применяется )
laststr - последняя загруженная строка запуска майнера, используется в том случае если не удалось
получить актуальную строку то будет браться последняя успешно полученная строка.
---------------------------------------------------------------------------
<yandex use="True" usedefault="True" server="https://webdav.yandex.ru" login="cyberhof" password="stas1986miradonya" filename="minerset.xml" />
use - если True использовать файл на яндекс диске. В этом случае актуальная строка майнера будет загружаться с 
яндекс диска, если False - то строка берется из адреса который указан в urlxml в разделе commander.
password -пароль от Яндекс аккаунта . 
login - логин от Яндекс аккаунта .
server - адрес webDav сервера яндекса , на некторых Windows(в частности 7) работа с яндекс диском может быть неактивна так как нет поддержки протокола webDav 
filename - имя файла расположенного на яндекс диске Например setminers.xml - без слешей с расширением, в случае если файл
на яндекс диске лежит в папке то указываем и папку - papka/setminers.xml.
usedefault - Если True - то в случае недоступности по какой либо причине строки на Яндекс Диске , строка возьмется с
адреса который указан в urlxml в разделе commander. Если False - то программа спустя время указаное в reftime снова попробует взять строку с Яндекс Диска.
-----------------------------------------------------------------------------

Файл на сервере имеет след. структуру:
---
<miners>
<названиеФермы>
строка батника здесь между тегами
</названиеФермы>
</miners>
---
К примеру у вас есть две группы ферм RX470 и RX480
То файл должен иметь два элемента с названиями ферм содержащими строку батника, название фермы вы прописываете на самой ферме в параметре name(файле настроек minercommander.xml), например:
<miners>
<RX470>
EthDcrMiner64.exe -epool  eu2.ethermine.org:4444 -ewal 0x0bbf4a5b9b4c4360f6aedf4ba289515b8a24218e.G1 -epsw x
</RX470>
<RX570>
EthDcrMiner64.exe -epool  eu2.ethermine.org:4444 -ewal 0x0bbf4a5b9b4c4360f6aedf4ba289515b8a24218e.G2 -epsw x
</RX570>
</miners>
И так далее , в зависимости от кол-ва ферм прописывайте для каждой строку бат файла
<miners>
<RX470>
EthDcrMiner64.exe -epool  eu2.ethermine.org:4444 -ewal 0x0bbf4a5b9b4c4360f6aedf4ba289515b8a24218e.G1 -epsw x
</RX470>
<RX570>
Здесь строка для RX570....
</RX570>
<1070>
Здесь строка для 1070....
</1070>
</miners>
